We already have a football match detail app with the following structure:

Match Detail Page includes:
- Top section: teams, league, date/time, stadium, last 5 form
- Tabs:
  1) Analiz
  2) Oranlar
  3) İstatistik
  4) Goller
  5) H2H
  6) Kadro

We have implemented the “Pre-Match Scenario” component inside the **Analiz** tab.
Now we want to extend the app with the following NEW FEATURES, fully aligned with the current structure and UI.

GLOBAL RULES:
- NO live / in-play features
- DARK UI only (same style as current cards)
- Mobile-first, iOS safe
- No betting CTA, no “guaranteed” language
- Use ONLY existing API-Football v3 data we already fetch:
  /fixtures, /predictions, /odds, /fixtures/lineups

UX:

- Clicking the match opens Match Detail as usual.

--------------------------------
FEATURE 2) “Confidence Skoru” inside PRE-MATCH SCENARIO (High priority)
--------------------------------
Inside the **Analiz tab → Pre-Match Scenario card**:

Add a new label near the scenario title:
- “Güven: Yüksek / Orta / Düşük”

Confidence logic:
- Base score = 60
- +20 if Dominance Index (DI) and Strength Gap (SG) point in same direction
- +10 if abs(DI) >= 0.25
- -15 if Chaos >= 67
Clamp 0–100

Label:
- 0–39: Düşük
- 40–69: Orta
- 70–100: Yüksek

This is NOT chaos. This is scenario reliability.

FEATURE 3) “What-if Senaryoları” (Analiz tab)
--------------------------------
Inside **Pre-Match Scenario card**, under the timeline:

Add 2–3 small toggle buttons:
- “Ev Sahibi Erken Gol Atarsa”
- “Deplasman Erken Gol Atarsa”
- “İlk Yarı 0-0 Biterse”

Behavior:
- No API call
- Recalculate scenario locally using simple offsets:
  - Home early goal: EARLY +0.12, BAL -0.05

UI:
- Smooth animation when toggled
- Clearly show “Simülasyon” state (small label)
